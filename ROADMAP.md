# ClawSandbox 路线图

## 项目历史

ClawSandbox 的初衷是解决一个简单的问题：在一台机器上部署多个隔离的 [OpenClaw](https://github.com/openclaw/openclaw) 实例，无需云基础设施或专用服务器。

### 开发时间线

1. **核心 CLI** — 基于 Docker 的容器生命周期管理（创建、启动、停止、销毁）
2. **Web 仪表盘** — 实时军团管理 UI，内嵌 noVNC 桌面
3. **版本管理** — CLI + Docker 镜像统一版本，GitHub Actions CI/CD 自动发布
4. **实例配置自动化** — 通过 CLI、API、Dashboard 自动完成 OpenClaw 初始化
5. **Dashboard-First 入门流程** — 全 WebUI 工作流：镜像构建、资产管理、军团运维

## 社区版 — 增长策略

### 价值主张

**"你已经有一台 Mac/Linux — 10 分钟部署一支 AI 员工团队 — 零额外开销。"**

### 为什么不用云？

云端部署路径存在根本性弊端，ClawSandbox 完全规避了这些问题：

- **开户与计费门槛** — 你需要注册云服务商（AWS / GCP / Azure），设置付款方式，在陌生的控制台里摸索，这一切都发生在你开始使用之前
- **持续月费** — 云实例每月都在产生费用，不管你是否在用。一台小型 VM 每月 $5–20，军团规模一上来开销就很可观
- **天然的中断性** — 云 VM 可能被迁移、终止、或触发账单限额。这种随时可能中断的环境与 OpenClaw 的"养成"体验背道而驰——你在培养一个长期陪伴的 AI 员工，不是启动一次性的计算任务
- **ClawSandbox 的替代方案** — 你的机器永远在线、永远属于你、零额外成本，龙虾的数据永久保存在你的磁盘上。养成关系不会被打断

### 用户漏斗

| 层级 | 描述 | 估算占比 |
|------|------|----------|
| 无关层 | 不关注 AI Agent | ~80% |
| 第一层 | 看别人用 OpenClaw，感兴趣 | ~20% |
| 第二层 | 自己部署并使用 OpenClaw | ~1% |
| 第三层 | 用 OpenClaw 解决实际问题 | ~0.01% |
| 第四层 | 用 OpenClaw 产生收入 | ~0.0001% |

**核心洞察：** ClawSandbox 同时打通两个转化瓶颈——第一层→第二层的部署门槛，以及第二层→第三层的军团能力缺失。

### 目标受众

**第一层用户 → 进入第二层（部署上手）— 19% 的未转化空间**

- 看过 OpenClaw / AI Agent 的 demo，觉得有意思
- 被部署门槛劝退（买设备 / 开云账号 / 命令行恐惧）
- **ClawSandbox 的价值：** 零额外成本，在自己电脑上安全地部署 OpenClaw，无需云账号、无需月费、数据完全在本地——10 分钟直接进入第二层

**第二层用户 → 进入第三层（解决实际问题）— 19.99% 的未转化空间**

- 已经体验过单个 OpenClaw，了解基本能力
- 但单个龙虾只能做简单任务，无法应对需要多角色协作的日常场景
- **ClawSandbox 的价值：** 一键部署 OpenClaw 军团，让多只龙虾各司其职（一只写代码、一只做调研、一只管社媒……），发挥军团协作能力，从"玩具"变成真正解决问题的工具——进入第三层

## 内部版 — 商业闭环

### 定位

小团队内部使用，基于稳定强大的 ClawFleet 构建 Human + OpenClaw 协作团队，通过真实商业实践验证闭环，最终产生收益。

### 核心思路

- 以一个稳定的 ClawFleet 为基础设施
- Human 负责决策与监督，OpenClaw 军团负责执行
- 通过真实商业场景积累经验，验证 AI 团队的商业价值
- 形成"投入（算力 + 管理） → 产出（商业收益）"的正循环

### 成功指标

**内部版：** 基于 ClawSandbox 军团验证的商业活动的 MRR — 代表小团队的经济收益
